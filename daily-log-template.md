# Daily Project Log

## How to Use This File

**Purpose:** Track what you work on each day for easy reference and weekly summaries.

**Update Frequency:** End of each day (or first thing next morning)

**Format:** Newest entries at top (reverse chronological)

## Tips for Daily Logging

### What to Track
✅ **DO track:**
- Completed deliverables (JIRA tickets, docs, decisions)
- Important meetings and outcomes
- Blockers encountered
- Communications sent (emails, Slack)
- Status changes
- Follow-up items with dates

❌ **DON'T track:**
- Every email you read
- Routine tasks (unless they're blockers)
- Minute-by-minute activity
- Personal tasks (unless impacting work)

### When to Update
- **End of day**: Best for fresh memory (5 minutes)
- **Next morning**: Good alternative if end-of-day doesn't work
- **Real-time**: Only if you have a system/tool that makes it easy

### Formatting Tips
- **Use emojis** for visual scanning (📋 🎯 📧 ⚠️ ✅ 🔵 ⏸️)
- **Start with project name** so you can find entries quickly
- **Link everything** (JIRA tickets, docs, Slack threads)
- **Be specific** about outcomes, not just activities
- **Note decisions** and who made them

### Common Emojis
- ✅ Complete
- 🔵 In Progress
- ⏸️ Paused
- 📋 JIRA/tickets
- 🎯 Goals/milestones
- 📧 Communication
- 🗓️ Meetings/scheduling
- ⚠️ Blockers
- 🔍 Investigation/research
- 📝 Documentation
- 🚀 Launches/releases

### Using for Weekly Summaries
At end of week, scan your daily log for:
1. Major accomplishments (copy to weekly update doc)
2. Meetings attended (note important ones)
3. Open blockers (carry forward to next week)
4. Follow-up items due next week

1. Create `daily-log-archive-YYYY-QX.md` file
2. Move old entries there
3. Keep current quarter in main file
4. Update this note with archive file names
