## Tips for Using This Format

### Column Definitions

**Week:** Date of the week (use Monday's date)
- Format: MM/DD/YYYY
- Helps track when work was done

**Who:** Your name or initials
- Useful if manager tracks multiple people in same doc
- Keep consistent

**Goal:** Short title (3-5 words)
- What is this work about?
- Examples: "Mobile Release", "VIP Dashboard Planning", "Sprint Planning"

**Description:** 1-2 sentences
- What did you do and what was the outcome?
- Include key metrics (ticket counts, percentages)
- Mention dependencies or blockers if relevant
- Examples:
  - "Product accepted 8 tickets for Tuesday 3/18 release including new dashboard and bug fixes"
  - "Created 5 JIRA tickets for VIP international expansion after stakeholder meeting confirmed scope"

**Priority:** Low / Medium / High
- High: Critical path items, blockers for others, leadership deliverables
- Medium: Regular sprint work, ticket creation, meetings
- Low: Documentation, process improvements, follow-ups

**Status:** Not Started / In Progress / Blocked / Done
- Not Started: Scheduled or planned but not begun
- In Progress: Actively working on it
- Blocked: Cannot proceed due to external dependency
- Done: Completed and delivered

**Asset/Doc:** Links to relevant resources
- JIRA epic links
- Confluence pages
- Google Docs
- Slack threads
- Use markdown link format: `[Text](url)`

---

### How to Update

**As You Work (Recommended):**
1. When you complete something significant, add a row
2. Keep rows in "Current Week" until you copy to manager's doc
3. At end of week, move rows to "Archive" section below

**End of Week:**
1. Copy all "Current Week" rows
2. Paste into your manager's status doc
3. Move rows from "Current Week" to "Archive" section
4. Add planned work to "Next Week"

**Weekly Cycle:**
```
Mon-Fri: Add rows as you complete work
Friday: Copy to manager's doc
Friday: Move to archive
Friday: Add next week's planned work
```

---

### Writing Good Descriptions

✅ **Good Examples:**
- "Product acceptance scheduled Monday 3/16 for Tuesday 3/17 release (9/11 tickets, 82% complete) - includes Dashboard updates, bug fixes, and new export feature"
- "Met with Finance stakeholders - approved VIP Dashboard metrics, requested export frequency control before pilot. Created PROJ-789 for enhancement."
- "Created comprehensive JIRA epic and 5 user stories for international expansion - all tickets in Ready for Refinement status"

❌ **Bad Examples:**
- "Worked on stuff" (too vague)
- "Had meetings" (no context or outcome)
- "Did JIRA tickets" (no specifics)

**Formula for Good Descriptions:**
`[Action verb] + [what you did] + [outcome/impact] + [optional: next step]`

Examples:
- "Completed product acceptance for mobile release - found date picker bug, logged PROJ-456 for fix in next sprint"
- "Published VIP timeline to Confluence including 10 tickets organized by dependency - ready for stakeholder review next week"

## Notes

**Customization Ideas:**
- Add "Team" column if you work across multiple teams
- Add "Quarter" or "Sprint" column for longer-term tracking
- Add "Impact" column to note business value
- Use different status labels if your team has different conventions

**Integration with Other Files:**
- Pull from your daily-log.md for details
- Link to dashboard.md for project context
- Reference when writing performance reviews

**Time Investment:**
- ~2 minutes per week to copy rows to manager's doc
- ~1 minute per significant work item to add row
- Total: ~10 minutes per week

**Pro Tips:**
- Keep descriptions executive-friendly (your manager may share up)
- Include metrics when possible (ticket counts, percentages, dates)
- Link to artifacts (makes it easy for manager to find details)
- Update regularly (easier than trying to remember at end of week)
